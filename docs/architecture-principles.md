# 重置版架构原则

这些是为了避免再次出现“看似修了一帧，实际把整个人画坏”的硬性规则。

## 1. 微表情不能切换整张人物图

眨眼、视线、眉毛和嘴型只改变对应图层。身体、衣服和主要轮廓在微表情期间保持同一份像素数据与同一锚点。

## 2. 所有视觉资源必须有统一坐标规范

角色母版、身体、头部、眼睛、眼皮、瞳孔、头发、辫子、饰品和服装层都使用固定画布、固定原点、固定缩放和命名规则。导入时自动检查尺寸与锚点，不依靠人工目测对齐。

## 3. 小动作程序驱动，大动作独立制作

- 呼吸、眨眼、瞳孔移动、轻微转头、发辫弹性使用连续参数驱动。
- 大幅转身、伸手、抚摸精灵和换姿势使用单独制作并校准的动作资源。
- 不把正面人物图随意旋转、拉伸或镜像来假装新的视角。

## 4. 旧版只作为诊断样本

旧工程用于比较功能和复现问题，不直接继承旧像素素材、整帧眨眼方法和堆叠式 UI。可复用的仅限经过审查的业务逻辑与数据格式。

## 5. 功能边界清晰

```text
桌宠表现层        角色、动作、气泡、面板
对话编排层        Provider 适配、工具调用、动作意图
上下文引擎        Token 预算、摘要、最近对话
长期记忆层        本地存储、检索、编辑、回收站、审计
电脑控制代理层    按需安装、三级权限、执行与回报
系统适配层        macOS 实现；未来替换为 Windows 实现
```

任何供应商专属的缓存、压缩或记忆能力都只能作为增强项，不能成为核心依赖。

## 6. 发布前必须实机验证

动画不仅检查单帧，还要检查循环视频中的基线跳动、比例漂移和轮廓闪烁。安装包必须在干净用户环境验证启动、退出、重开、本地数据和系统权限提示。
