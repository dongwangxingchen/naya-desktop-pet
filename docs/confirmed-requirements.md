# 已确认需求

本文只记录已经由用户明确确认的方向，未确认的细节不在这里擅自定案。

## 桌宠本体

- 新版本不沿用旧像素造型，使用已经认可的清爽、偏白肤色、非像素角色方向。
- 默认姿势应慵懒、自然，不能像站桩一样呆。
- 角色始终保持可交互，不进入无法对话的长期睡眠状态。
- 大型待机动作以约 10～20 分钟一次为保守起点。
- 眼睛可以偶尔看向鼠标，但不能机械地一直追踪。
- 夜晚增加困倦专属动作，也可以在未来支持换装。

## 猴头菇精灵

- 不是常驻伙伴。
- 偶尔从屏幕边缘上下晃动飞入。
- 可以停在纳雅肩上，也可以飞到手中，由纳雅轻柔抚摸。
- 先制作少量可靠动作，不在第一阶段过度扩展。

## AI 与对话

- AI 功能完全可选；不配置接口也能正常使用桌宠。
- 设置项包含 URL、API Key、连接检测、模型刷新、模型选择与保存。
- 兼容 OpenAI、DeepSeek、Anthropic Claude、Gemini、OpenAI 兼容接口和常见中转站。
- 优先自动识别协议；识别失败时允许手动选择。
- AI 通过结构化指令选择合适的表情、视线和动作。
- 桌面只显示最新回复气泡；完整聊天记录保存在管理面板。
- 气泡根据角色和屏幕边缘自动换方向，不能遮住角色或超出屏幕。

## 上下文与记忆

- 区分提示词缓存、当前会话压缩和跨会话长期记忆。
- 重启后仍能继续记得用户。
- 会话压缩结束后立即继续对话，不强制等待固定时间。
- 压缩期间显示自然的整理思绪提示，并播放专属动作。
- 长期记忆有独立界面，用户可以搜索、修改和删除。
- 删除的记忆先进入回收站，可以恢复。
- 固定角色人格为只读内容，AI 无权自行删除。
- 所有聊天、摘要、长期记忆和审计记录保存在本地。

## 电脑控制

- 电脑控制模块默认不安装、默认关闭。
- 用户配置 AI 并主动选择电脑控制权限后，才提示按需下载成熟的本地能力包。
- 第一档：只读访问，禁止修改；这是默认档位。
- 第二档：可以修改，但每次执行前说明内容并请求批准。
- 第三档：完全允许，无需逐次批准，并跨重启保持到用户主动更改。
- 目标覆盖终端、文件、鼠标、键盘、浏览器和图形应用。
- 优先整合经过社区验证的开源运行时，不制作廉价的自动点击封装。

## 平台

- 第一版优先完整支持当前 macOS 系统。
- 为 Windows 保留清晰的系统适配边界，后续再实现 Windows 版本。

## 工具与制作方式

- 不购买 Live2D Cubism、Spine 等付费绑定工具。
- 不聘请专业绑定师，角色绑定由项目自行完成。
- Godot 负责透明窗口、桌宠状态机、UI、AI 和系统能力；角色绑定编辑器与运行时必须使用免费或开源方案。
- 允许先隔离测试 `nijigenerate`、Blender 和 Godot 原生分层，再按可复现结果选择生产管线。
- 可以慢慢迭代，但每次只扩大一个经过验证的动作范围。
- 标准分层是一次性制作角色母版，不等于旧版那种把好看的局部反复抠到不同坏帧上拼接。
