# 小纳雅桌宠重置版工作规则

## 产品边界

- 这是重置版仓库。旧像素工程只能用于复现问题和迁移经过审查的业务逻辑，不能把旧素材重新混入新角色。
- 桌宠基础体验必须在没有 AI、没有 API Key、没有电脑控制模块时正常运行。
- 第一阶段以 macOS 为目标，同时保持系统适配接口可替换。
- 角色生产只使用免费或开源工具；Godot 固定负责桌宠产品外壳，绑定编辑器与角色运行时必须先通过隔离技术样机，不能过早锁死。
- `nijigenerate`、`nicxlive`、Blender 或第三方 Godot 插件均不得未经稳定性、许可证和可回退验证就成为生产依赖。

## 角色与动画

- 不允许用多张独立生成的整身图片制作眨眼、视线或嘴型。
- 不允许把旧图中“看起来好的部位”抠出后拼到新图上。
- 生成或修改角色资源时必须附带已确认的角色参考图，并保持同一母版、比例、线条、配色和饰品位置。
- 微表情必须使用固定身体与分层面部；所有图层必须通过自动尺寸和锚点检查。
- 大动作可以使用独立姿态，但必须检查进入、退出和循环时的比例、基线与身份一致性。

## 数据与安全

- API Key 不得写入仓库、日志或普通配置文件；macOS 使用 Keychain。
- 聊天、摘要和长期记忆默认仅保存在本地。
- 电脑控制模块按需安装且默认关闭；默认权限为只读。
- 任何具备修改能力的操作都必须服从用户选择的权限档位。

## 开源组件

- 优先采用成熟、活跃且许可证清晰的开源组件。
- 引入前记录项目地址、固定版本、许可证、用途和替代方案。
- 不直接复制来源不明的代码，不删除原作者版权或许可证声明。

## 发布

- 不发布旧像素安装包作为重置版 Release。
- Release 必须满足 README 中的首包验收门槛，并保留验证记录。
- 未签名或未公证的 macOS 构建必须明确标为测试包，不能伪装成正式稳定版。
